# Chrome Bookmark List - 外部仕様書

## 概要

Chrome Bookmark Listは、Chromeブラウザの新しいタブでブックマークをフォルダ別に整理して表示し、履歴も検索できるChrome拡張機能です。ユーザーはブックマークを視覚的に管理し、素早くアクセスでき、さらに履歴機能で過去の閲覧記録も確認できます。

## 主要機能

### 1. ブックマーク表示機能
- **目的**: Chromeのブックマークをフォルダ構造に従って一覧表示
- **動作**: 新しいタブを開くと、ブックマークが自動的にフォルダ別に表示される
- **表示内容**:
  - フォルダ名とフォルダ内のブックマーク数
  - 各ブックマークのタイトル、URL、Favicon
  - 階層構造（親フォルダ → 子フォルダ → 孫フォルダ）

### 2. フォルダ展開・折りたたみ機能
- **目的**: 大量のブックマークを整理して見やすくする
- **操作方法**:
  - サブフォルダがあるフォルダ: フォルダヘッダーをクリックしてサブフォルダの表示/非表示を切り替え
  - ブックマークのみのフォルダ: フォルダヘッダーをクリックしてブックマークリストの表示/非表示を切り替え
- **視覚的表示**:
  - 📁（折りたたみ状態）/ 📂（展開状態）のアイコン表示
  - サブフォルダは薄紫色の背景で区別

### 3. ブックマークアクセス機能
- **目的**: ブックマークへの素早いアクセス
- **操作方法**: ブックマークリンクをクリック
- **動作**: 新しいタブでブックマークのページが開く

### 4. リアルタイム検索機能
- **目的**: 大量のブックマークから目的のものを素早く見つける
- **操作方法**: ページ上部の検索ボックスに文字を入力
- **検索対象**: ブックマークのタイトルとURL
- **動作**:
  - 入力に応じてリアルタイムでフィルタリング
  - 該当するブックマークを含むフォルダのみ表示
  - 検索時は関連フォルダが自動展開

### 5. Favicon表示機能
- **目的**: ブックマークの視覚的な識別性向上
- **動作**: 
  - Chrome の `_favicon` API（`chrome.runtime.getURL('/_favicon/...')`）によるローカル取得
  - favicon が未保有の場合は Chrome の標準アイコン（地球儀）を表示
- **セキュリティ重視**: 外部APIサービスを使用せず、ホスト名の流出を防止

### 6. タブナビゲーション機能
- **目的**: ブックマークと履歴を1つのページ内でタブ切り替えして表示する
- **操作方法**:
  - ヘッダー右側のタブ（📚 ブックマーク / 🕐 最近の履歴 / 🗂 最近閉じたタブ / 📅 カレンダー）をクリックして表示を切り替える（タイトルと同じ行に配置）
- **動作**:
  - 選択中のタブはアクセントカラーでハイライト表示
  - 「最近の履歴」「最近閉じたタブ」「カレンダー」タブは選択時にデータを読み込む（遅延読み込み）
  - ブックマーク・最近の履歴・カレンダーの各タブはパネル上部に専用の検索バーを持つ（最近閉じたタブは検索なし。§9 参照）。タブを切り替えてもヘッダー・タブの位置は変わらない
- **アクセシビリティ**: `role="tablist"` / `role="tab"` / `role="tabpanel"` と `aria-selected` を付与

### 7. 最近の履歴タブ
- **目的**: 過去の閲覧履歴を確認し、再アクセスを容易にする
- **操作方法**:
  - 「🕐 最近の履歴」タブをクリックして表示する
- **表示内容**:
  - 過去7日間の履歴を最大50件、カード形式のグリッドで表示
  - 各履歴項目のタイトル、URL、Favicon
  - 最終訪問日時、訪問回数
  - 履歴項目のタイトルをクリックすると新しいタブで開く
- **検索機能**:
  - パネル上部の検索バーで履歴をフィルタリング
  - タイトルとURLの両方を検索対象とする
  - 大文字小文字を区別しない検索
  - リアルタイム検索（入力と同時にフィルタリング）

### 8. カレンダー履歴タブ
- **目的**: 当月分の閲覧履歴を日毎かつ単位時間１時間単位で確認し、履歴情報の確認を容易にする
- **操作方法**:
  - 「📅 カレンダー」タブをクリックして表示する
- **カレンダー表示**:
  - 左側に当月のカレンダーを表示（幅380px）
  - 履歴がある日付には青い点のインジケーター表示
  - 今日の日付はアクセントカラーでハイライト
  - 選択中の日付はアクセントグラデーション背景で表示
  - 前月・次月ボタンで月を切り替え可能
- **タイムライン表示**:
  - 右側に選択した日の履歴を時間単位（1時間ごと）で表示
  - 時間単位のナビゲーションリンク（スティッキー表示）で素早く該当時間にジャンプ
  - 各時間帯セクションは古い順（時系列順）で表示
  - 日付選択時、タイムラインのスクロール位置は自動的にトップに戻る
  - 履歴項目のタイトルをクリックすると新しいタブで開く
- **ドメイン統計表示**:
  - 各時間帯セクションの下に訪問ドメインの統計を表示
  - ドメインごとの訪問回数を訪問数の多い順にソート
  - 各ドメインのFaviconアイコンを表示（取得失敗時は🌐アイコン）
  - ドメイン名と訪問回数をバッジ形式で表示
- **履歴アイテム表示**:
  - 各履歴項目のタイトル、URL、Favicon
  - 訪問時刻（HH:MM:SS形式）
  - 訪問回数
  - URLは長い場合に省略表記（...）で表示
- **検索機能**:
  - パネル上部の検索バーで履歴をフィルタリング
  - タイトルとURLの両方を検索対象とする
  - 大文字小文字を区別しない検索
  - リアルタイム検索（入力と同時にフィルタリング）
  - 検索時は時間単位のグルーピングを維持

### 9. 最近閉じたタブ機能
- **目的**: 誤って閉じたタブや再度確認したいタブを素早く復元する
- **操作方法**:
  - 「🗂 最近閉じたタブ」タブをクリックして表示する
- **表示内容**:
  - `chrome.sessions.getRecentlyClosed()` による最大約25件の一覧表示
  - 単体タブのみ対象（ウィンドウとして閉じたものは対象外）
  - 自拡張の New Tab ページは除外
  - 同一 URL は最新（最も最近閉じたもの）のみ表示（重複排除）
  - 各タブのタイトル、ドメイン、favicon、閉じた日時（lastModified）を表示
- **復元動作**:
  - タブ項目をクリックすると `chrome.sessions.restore(sessionId)` により元の位置へ復元
  - 復元後は一覧を自動再読み込みする（復元した項目は一覧から消える）
- **検索機能**: 検索フィルタは無し（閉じたタブ件数は最大約25件程度のため）

### 10. ドラッグ&ドロップによるブックマーク移動機能
- **目的**: ブックマークを直感的にフォルダ間で移動する
- **操作方法**:
  - ブックマークアイテムをマウスでドラッグ
  - 移動先フォルダのヘッダー部分にドロップ
  - 複数選択したブックマークを掴んでドラッグすると、選択中の全アイテムを一括で移動できる
- **視覚的フィードバック**:
  - ドラッグ中: ブックマークに「dragging」クラスが適用され見た目変化
  - ドロップターゲット: フォルダヘッダーがハイライト表示
  - ドロップインジケーター: 「ここにドロップ」のテキスト表示
- **制限事項**:
  - 同じフォルダ内での移動は無効
  - 一括移動時、選択中の項目のうち既に移動先フォルダにあるものは移動対象から除外される
- **動作**: Chrome Bookmarks APIを使用してブックマークを実際に移動し、UIを自動更新

### 11. キーボードショートカット機能
- **目的**: マウスを使わずにブックマークを操作できるようにする
- **対応ショートカット**:
  - `↑` / `↓`: フォーカスをブックマーク・フォルダ間で移動
  - `Enter`: フォーカスしているブックマークを開く / フォルダを展開・折りたたみ
  - `Delete` / `Backspace`: フォーカスしているブックマークを削除（確認ダイアログを表示）
  - `F2`: フォーカスしているブックマークを編集
  - `Cmd/Ctrl + F`: 現在表示中のタブの検索入力欄にフォーカス（検索欄の無いタブ〔最近閉じたタブ〕ではブラウザ標準の検索に委ねる）
  - `Cmd/Ctrl + Z`: 直前の破壊的操作を取り消す（Undo）
  - `Shift + F10`: フォーカスしている項目のコンテキストメニューを表示
  - `?`: キーボードショートカット一覧ヘルプを表示
  - `Esc`: ダイアログやコンテキストメニューを閉じる
- **入力欄での挙動**: 検索入力欄など編集要素にフォーカスがある場合、矢印キーや Delete などの単独キーショートカットは抑制し、標準動作を優先する
- **アクセシビリティ**: ブックマーク項目とフォルダヘッダは `tabindex="0"` と `role="treeitem"` を付与してフォーカス可能になっている

## ユーザーインターフェース

### レイアウト
- **タブナビゲーション**: ヘッダーのタイトルと同じ行に配置したピル型タブでブックマーク／最近の履歴／最近閉じたタブ／カレンダーを切り替え
- **グリッドレイアウト**: ブックマークフォルダ・履歴項目がカード形式で配置
- **レスポンシブデザイン**: 画面サイズに応じて最適化（モバイルでは1カラム）
- **モダンなUI**: カードのホバーエフェクト、シャドウ効果、アクセントグラデーション
- **ダークモード対応**: OSの配色設定（prefers-color-scheme）に連動してライト／ダークを自動切り替え

### 操作性
- **直感的操作**: クリックベースのシンプルな操作
- **視覚的フィードバック**: ホバー効果、アイコン変化
- **高速レスポンス**: 軽量設計による快適な操作感

## 対応環境

### 必要条件
- **ブラウザ**: Google Chrome（Manifest V3対応版）
- **権限**:
  - `bookmarks`: ブックマークデータへのアクセス
  - `history`: 履歴データへのアクセス
  - `favicon`: Chrome内部のfavicon APIへのアクセス
  - `sessions`: 最近閉じたタブの取得と復元（`chrome.sessions` API）へのアクセス
  - `tabs`: `chrome.sessions` が返す閉じたタブの URL・タイトルの読み取りに必要
    （この権限が無いと Chrome が URL/タイトルを除去するため一覧を表示できない。
    開いているタブの監視には使用しない）

### パフォーマンス
- **起動時間**: 高速な初期読み込み
- **メモリ使用量**: 軽量設計
- **レスポンス性**: スムーズなUI操作

## 制限事項

1. **Chrome専用**: Chrome拡張機能のため、他ブラウザでは使用不可
2. **権限必須**: ブックマーク、履歴、favicon、sessions、tabsの権限が必要
3. **表示制限**: 極めて大量のブックマーク（数千個以上）では表示速度が低下する可能性
4. **履歴期間制限**:
   - 最近の履歴タブ: 過去7日間、最大50件
   - カレンダー履歴タブ: 当月分、最大10,000件
5. **カレンダー履歴の制限**: 月ごとに最大10,000件の履歴を読み込むため、非常に多くの履歴がある場合は一部が表示されない可能性

## セキュリティ

- **プライバシー保護**: ブックマーク・履歴データはローカルでのみ処理、外部送信なし
- **ホスト名保護**: 内部ネットワークのホスト名が外部サービスに送信されることを防止
- **外部依存なし**: 外部APIサービスを使用せず、完全に自己完結
- **権限最小化**: 必要最小限の権限のみ要求
- **安全な実装**: XSS対策としてHTMLエスケープ処理を実装
